## Variant: Tool-dense

### Design stance
Dashboards are tools first. Data density and keyboard-friendly interaction patterns for users who manage many vaults.

### Key choices
- **Layout:** Tabbed navigation (Overview / Vaults / Activity / Analytics) with a stat bar, filter chips, and a full data table
- **Typography:** Smaller base (13px), monospace for numbers, uppercase labels. Maximum scanability.
- **Interaction:** Click column headers to sort ascending/descending. Filter by status. Search by name. Click a row to see a detail bar at the bottom with Deposit/Withdraw actions.
- **Data:** 8 vaults visible in one viewport. Search + filtering on all rows.

### Trade-offs
- **Strong at:** Power users managing 10+ vaults. Quick scanning, sorting, filtering.
- **Weak at:** First emotional impression — functional, not inspiring. Overwhelming for casual users.

### Best for
- Treasury managers, institutional users, power users.
- Internal operations dashboard.
- The "show me everything and let me act on it" view.

# Swarm Simulation Reference

## Overview

The swarm simulation runs 100 autonomous agents across 7 archetypes making weekly decisions against the live protocol over 320 ticks (~6 years). Agents interact with VaultNFT, BtcToken, PerpetualVault, VolatilityPool, and a SimCurvePool AMM.

## Invariants

### vBTC Ratio Bounds (Per-Tick)

**Location:** `test/SwarmSimulation.sim.t.sol::_checkRatioInvariants(uint256 tick)`  
**Library:** `src/assertions/SwarmInvariants.sol::invariant_vbtcRatioBounds(uint256 vbtcRatio)`

The vBTC/WBTC exchange ratio must remain within the protocol-defined economic bounds at all times once the Curve pool is initialized:

- **Ceiling:** `vbtcRatio <= 1.0e18` (1.0 = full peg)
- **Floor:** `vbtcRatio >= 0.5e18` (0.5 = maximum discount)

These bounds are enforced by:

1. **AMM-level enforcement:** `SimCurvePool` rejects swaps and liquidity additions that would push the ratio outside `[0.5, 1.0]` via `RatioBoundsExceeded`.
2. **Per-tick simulation assertion:** `_checkRatioInvariants()` runs every tick (not just every 50 ticks) once the pool is initialized. If the ratio breaches either bound, the simulation halts immediately with a clear error message:
   - `"Tick N: Ratio ceiling breached. vbtcRatio=X"`
   - `"Tick N: Ratio floor breached. vbtcRatio=X"`
3. **Standalone fuzz invariant:** `SwarmInvariants.invariant_vbtcRatioBounds()` can be called from handler-based fuzz tests for property-based verification independent of the swarm simulation.

**Rationale:** The vBTC token represents a claim on BTC collateral within a vault. If the ratio exceeded 1.0, vBTC would be worth more than the underlying BTC (arbitrage opportunity). If it fell below 0.5, the discount would be so severe that protocol solvency concerns would arise. The [0.5, 1.0] range is the protocol's economically viable operating band.

### Other Invariants

The full invariant suite runs every 50 ticks:

| Invariant | Check | Location |
|-----------|-------|----------|
| Vault solvency | `wbtc.balanceOf(vault) >= matchPool` | `_checkInvariants()` |
| Perp vault solvency | `vBTC.balanceOf(perp) >= sum(position payouts)` | `SwarmInvariants.checkPerpVaultSolvency()` |
| Vol pool solvency | `vBTC.balanceOf(volPool) >= longPoolAssets + shortPoolAssets` | `SwarmInvariants.checkVolPoolSolvency()` |

## Execution

### Running a Single Simulation

```bash
cd contracts/simulation
forge test --match-test test_swarm -vvv --gas-limit 999999999999
```

Override duration via `SIMULATION_WEEKS` (default 320).
Override seed via `SIMULATION_SEED` (default 42).

### Running All 3 Seeds

```bash
cd contracts/simulation
bash run_3_sims.sh
```

Seeds: 42, 123, 777. Each runs 320 ticks. Results archived to `sim_results/sim_N_seed_X/`.
